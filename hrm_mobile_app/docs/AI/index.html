✨ Architecture Highlights

📱 Offline Support SQLite queue + exponential backoff retry (1s→2s→4s→8s)
🔐 Secure by Default JWT auth, PII masking, encryption at rest, rate limiting
⚡ Fast Response 2-4 seconds E2E latency (happy path), p95: 2.5s
🤖 AI-Powered Dify RAG + LLM, confidence scoring, low-confidence escalation
👤 Human Handoff Smart escalation, Slack integration, agent chat session
📊 Observable Structured logging, metrics, distributed tracing, SLA monitoring
🛡️ Error Resilient 15+ edge cases handled, fallback responses, graceful degradation
📈 Scalable Horizontal scaling, connection pooling, async processing

🏛️ System Architecture (7 Layers)

1️⃣ Flutter Frontend

Responsibilities: User input, validation, offline support, state tracking

Technology: Flutter, SQLite/Hive, JWT

Latency: ~70ms per operation

2️⃣ NestJS Backend Gateway

Responsibilities: Auth, rate limiting, message persistence, webhook formatting

Technology: NestJS, PostgreSQL, JWT middleware

Latency: ~26ms per operation

3️⃣ n8n Orchestrator

Responsibilities: Context management, token optimization, intent classification, error handling

Technology: n8n, workflow automation

Latency: ~55ms per operation

4️⃣ Dify (AI Brain)

Responsibilities: LLM inference, RAG queries, confidence scoring, refusal handling

Technology: Dify, Claude/GPT, RAG knowledge base

Latency: 1.5-3 seconds ⭐ (bottleneck)

5️⃣ PostgreSQL Integration

Responsibilities: Message history, conversation context, orders, inventory

Technology: PostgreSQL, connection pooling

Latency: ~25ms per query

6️⃣ Human Handoff

Responsibilities: Escalation logic, Slack notifications, agent chat routing

Technology: Slack API, support ticket system

Latency: ~200-500ms for notification

7️⃣ Observability

Responsibilities: Logging, metrics, alerting, distributed tracing

Technology: ELK Stack, Prometheus, Grafana, Jaeger

Metrics: p50/p95/p99 latency, error rate, SLA tracking

⏱️ Performance Targets

E2E Latency (Happy Path)

Flutter Input (70ms) → NestJS (26ms) → n8n (55ms) → Dify (1.5-3s) → Response (150ms)

Total: 2.0-2.5 seconds (p50) | 2.5s (p95) | 4.0s (p99)

SLA Targets

  • p50 (median): < 2.0 seconds
  • p95: < 2.5 seconds
  • p99: < 4.0 seconds
  • Error Rate: < 1%
  • Availability: 99.5%

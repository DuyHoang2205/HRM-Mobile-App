Edge Cases & Error Handling

Comprehensive handling for all failure scenarios and boundary conditions.

🔵 Flutter Frontend Edge Cases

  • Invalid File Type: Block Voice/Video/Exe files at client → Show error toast → No API call made
  • Offline Mode: Save to SQLite (Hive) → Queue message for retry → Auto-sync when online → Exponential backoff (1s → 2s → 4s → 8s)
  • Message State Tracking: pending → sent → delivered → read (UI updates)
  • Push Notification Handling: Receive response notification → Open chat screen → Mark as read
  • Large File Upload: Show progress bar → Resume on network interruption

🔴 NestJS Gateway Edge Cases

  • JWT Expired: Validate JWT → If expired, request refresh token → Retry with new token → If both fail: 401 Unauthorized
  • Rate Limiting: 100 requests/minute per user → On exceeded: return 429 Too Many Requests → Client-side retry with exponential backoff
  • Deduplication: If same message_id arrives twice (webhook retry) → Idempotent: don't create duplicate in DB
  • Database Connection Failure: Fail fast, return 503 Service Unavailable → Alert ops team
  • Webhook Timeout to n8n: Retry up to 3 times → If still fails: queue in dead-letter queue (DLQ) → Manual replay later

🟠 n8n Orchestrator Edge Cases

  • Context Window Overflow: Count tokens of conversation history → If > 4K: truncate old messages → Keep system prompt + last 5-10 messages
  • Missing Conversation Context: Session ID not found in DB → Create new conversation context → Initialize with user profile data
  • Intent Classification Ambiguity: If multiple intents detected → Ask clarification: "You want to check order or ask about size?"
  • Dify API Timeout: Set timeout to 5 seconds → If exceeded: cancel request → Send fallback message: "Sorry, please wait..." → Escalate to human
  • Webhook Validation Failure: Verify signature + payload format → If invalid: log security event + reject (400 Bad Request)

🟣 Dify (AI) Edge Cases

  • Contradictory User Data: Input: "Height 1m8, Weight 40kg" (unrealistic) → Detected via validation rules → Ask: "Please confirm your weight?"
  • Context Switching: User was asking about áo (shirt) → Suddenly switches to "Tôi muốn đổi quần cũ" (return old pants) → Detect intent change → Exit previous flow → Process new intent
  • Refusal Cases: Out-of-scope questions (gold prices, politics, etc.) → Dify configured to politely refuse: "I can only help with clothing items"
  • Low Confidence Score: If confidence < 60% → Suggest escalation: "I'm not sure, let me connect you with an agent" → Trigger human handoff
  • Hallucination Prevention: If Dify generates info not in RAG KB → Confidence score auto-drops → Triggers refusal or escalation
  • Language Mix: User sends Viet + English mix → Dify configured to handle → Respond in same language as user's primary

🐘 PostgreSQL & Integration Edge Cases

  • Out of Stock (Inventory Check): n8n queries: "SELECT * FROM products WHERE product_id = ?" → If stock = 0 → Call Dify again: "Suggest alternative colors/sizes" → User sees recommendations
  • Order Not Found: User asks for order #12345 → DB query returns 0 rows → Dify: "I couldn't find that order. Can you provide more details?"
  • Concurrent Updates: While user is chatting, inventory updates in parallel → Use versioning or pessimistic locking → Ensure consistency
  • Database Slow Query: Complex JOIN takes > 2s → Query timeout occurs → Fallback: "Data loading slow, please try again" → Escalate

👤 Human Handoff Edge Cases

  • No Agents Available: All agents busy → Queue customer → Estimated wait time: "5 agents ahead of you" → Customer can leave message
  • Agent Disconnect: Agent session drops mid-conversation → Detect disconnect → Automatically queue customer again → New agent takes over
  • Escalation Loop Prevention: If escalated message fails to reach Slack (Slack API down) → Retry up to 3x → Send backup email to support team
  • Conversation History Transfer: Entire chat history (AI + user) sent to agent → Agent can see what AI already tried → No duplicate questions
🎯 Overall Strategy:

Fail gracefully → Provide fallback response → Escalate to human → Log everything for analysis. Never return blank/error to user.

🔐 Security: JWT auth (15-min refresh), PII masking in logs, encryption at rest (pgcrypto), HTTPS only

📊 Monitoring: ELK Stack (structured logs), Datadog (metrics), distributed tracing with trace_id

🔄 Deployment: Canary deploy, auto-rollback on error spike, read replicas for PostgreSQL
